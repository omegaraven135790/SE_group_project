In our project we planned to have a total of 3 sprints with each lasting approximately one week and  with tasks always proceeding through its relevant states(analysis,
design,implementation) and then finally progressing to review where another group member checks the work for bugs or any faults and if all is fine
it is moved to done state. For each state generally 2-3 sentences are written to provide a brief description of what was done. At the end
of each sprint we evaluate how it went and what we learned from it.

The plan for the first sprint:
Initial analysis and planning done, diagrams such as the use case diagram and class diagram are completed along with the project vision and justification document and development has started on the wireframe for the features.



The plan for the second sprint:
focus on implementation of the parts of the wireframe already completed and the development for the prototype is started and remaining features that have not already been designed in the wireframe have their wireframe and prototype done simultaneously 
preparing final report and slides, update burndown chart 

The plan for the third sprint:
The prototype should be done or near completion, writing test cases is started and completed before the end of this phase based on the prototype and user feedback is collected and any necessary changes to the prototype are done based on the user feedback.
Burn down charts and other final documents are updated and double checked.
